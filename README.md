# Sum over Clifford Groups

**Decomposing multi-controlled Z gates as weighted sums of real diagonal Clifford group elements via linear programming.**

---

## Overview

This project investigates the *stabilizer decomposition* of multi-controlled Z (C^k Z) gates.
Given a target gate whose diagonal unitary has entries in {+1, −1}, the goal is to express it as a
non-negative linear combination of elements from the **real diagonal Clifford group**:

```
U_target = Σ cᵢ Dᵢ,    cᵢ ≥ 0,    Dᵢ ∈ real diagonal Clifford group
```

and to find the decomposition that **minimises the total weight** `Σ cᵢ` (the *stabilizer extent* of the gate).
This quantity squared, `(Σ cᵢ)²`, lower-bounds the classical simulation cost of the gate in the stabilizer formalism.

The problem is cast as a **linear programme (LP)** and solved with industrial-strength LP solvers
(HiGHS, CVXPY/CLARABEL, and GLPK), scaled up to 6 qubits — covering gates with up to 2^6 = 64
diagonal entries.

---

## Background

### Clifford groups and fault-tolerant quantum computing

The *n*-qubit Clifford group is the set of unitary operations that map Pauli operators to Pauli operators
under conjugation. Clifford circuits are efficiently classically simulable (Gottesman–Knill theorem),
but universal quantum computation requires non-Clifford resources (e.g., T gates or multi-controlled Z gates).

A key question in resource theory is: **how many Clifford building blocks does a non-Clifford gate require?**
The *stabilizer rank* and *stabilizer extent* are measures of this cost.

### Real diagonal Clifford subgroup

The **real diagonal Clifford group** on *n* qubits is the subgroup of diagonal Clifford unitaries
whose diagonal entries are all real (i.e., ±1). It is generated by:
- The phase gate **P** (and its powers) on each qubit
- The **CZ** gate (and its square) on every pair of qubits

This gives groups of sizes 2, 32, 64, 1024, 32768, and 2097152 for 1–6 qubits respectively.

### The decomposition LP

For a target gate with diagonal `b ∈ {±1}^(2^n)`, let `A` be the matrix whose columns are the
diagonals of all real diagonal Clifford group elements (and their negatives). The decomposition is:

```
minimise    1ᵀ x
subject to  A x = b
            x ≥ 0
```

The optimal objective value is the *stabilizer extent* of the target gate.

## Installation

### Option A — Local (Python ≥ 3.10)

```bash
# 1. Clone the repository
git clone <repo-url>
cd Sum-over-clifford-groups

# 2. Create and activate a virtual environment
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate

# 3. Install Python dependencies
pip install -r requirements.txt

# 4. (Optional) Install python-quaec — only needed to regenerate full Clifford groups
pip install git+https://github.com/cgranade/python-quaec.git
# Fix a Python 3.10+ compatibility issue in qecc:
sed -i 's/from collections import Sequence/from collections.abc import Sequence/' \
    .venv/lib/python*/site-packages/qecc/paulicollections.py

# 5. Add the project root to PYTHONPATH
export PYTHONPATH=$PYTHONPATH:$(pwd)
```

### Option B — Docker (includes GLPK and all dependencies)

```bash
docker compose up --build
```

The container is based on Python 3.11 and compiles GLPK from source.

## Usage

### Step 0 — Generate datasets (optional)

Pre-computed datasets are already in `datasets/real_diagonals/`. You only need to regenerate them if you
want to verify or extend to more qubits.

```bash
# Run from the project root
python generate_clifford_groups/real_diagonal_clifford_groups.py
```

Edit the `__main__` block to select which qubit counts to generate (1–6).
Note: 6-qubit generation takes significant time and memory (~2 M elements).

### Step 1 — Run the HiGHS solver (recommended)

```bash
python solvers/highs_real_diagonal_clifford.py
```

Supported target gates (edit the `__main__` block to select):

| `target` argument    | Gate              | Qubits |
|----------------------|-------------------|--------|
| `'cz'`               | CZ                | 2      |
| `'ccz'`              | CCZ (Toffoli-Z)   | 3      |
| `'cccz'`             | CCCZ              | 4      |
| `'ccccz'`            | CCCCZ             | 5      |
| `'cccccz'`           | CCCCCZ            | 6      |
| `'cz_tensor_cz'`     | CZ ⊗ CZ           | 4      |
| `'ccz_tensor_cz'`    | CCZ ⊗ CZ          | 5      |
| `'ccz_tensor_ccz'`   | CCZ ⊗ CCZ         | 6      |
| `'circuit_cz_4'`     | Circuit of CZ gates | 4    |
| `'circuit_cz_5'`     | Circuit of CZ gates | 5    |
| `'circuit_cz_6'`     | Circuit of CZ gates | 6    |

You can also call the solver programmatically:

```python
from solvers.highs_real_diagonal_clifford import run_highs_solver

run_highs_solver(target='ccz', num_threads=4)
```

Results are written to `highs_solution_<num_threads>/solution_summary_<target>.txt`.

### Step 2 — Run the CVXPY solver (alternative)

```bash
python solvers/cvx_real_diagonal_clifford.py
```

Edit the `__main__` block or call directly:

```python
from solvers.cvx_real_diagonal_clifford import run_cvxpy_solver

run_cvxpy_solver(target='ccz')
```

Results are written to `cvxpy_final_solutions/solution_summary_<target>.txt`.

### Step 3 — Run the GLPK C solver (high-performance)

Compile and run from the project root:

```bash
gcc -O2 -o glpk_solver solvers/glpk_real_diagonals.c -lglpk -lm
./glpk_solver
```

Edit the global variables at the top of `glpk_real_diagonals.c` to select the number of qubits
and the path to the corresponding dataset file.
